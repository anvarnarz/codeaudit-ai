Create `packages/audit-engine/src/tools/exec-command-tool.ts` with a `createExecCommandTool(repoPath: string, timeoutMs?: number)` factory function that returns a Vercel AI SDK `tool()` definition.

**Tool schema (Zod):**
```typescript
z.object({
  command: z.string().describe("The shell command to run (e.g., 'find', 'grep', 'cat', 'head', 'wc', 'git')"),
  args: z.array(z.string()).describe("Command arguments as an array"),
})
```

**Sandbox enforcement (PRF-14) â€” validate BEFORE executing:**

1. **Command allowlist** â€” only these commands are permitted:
   `find`, `grep`, `egrep`, `cat`, `head`, `tail`, `wc`, `ls`, `tree`, `file`, `stat`, `du`, `sort`, `uniq`, `awk`, `sed` (without -i), `git`, `bash`, `sh`, `npm`, `npx`, `pip`, `pip3`, `cargo`, `go`, `python`, `python3`, `ruby`, `php`, `java`, `javac`, `dotnet`, `swift`, `cloc`, `tokei`, `scc`, `jscpd`

2. **Dangerous pattern blocklist** â€” reject if ANY arg matches:
   - Write/delete: `rm `, `mv `, `cp `, `mkdir`, `rmdir`, `chmod`, `chown`, `> `, `>> `, `tee `, `touch `
   - Network: `curl `, `wget `, `fetch `, `nc `, `ssh `, `scp `
   - Package install: `npm install`, `npm ci`, `yarn add`, `pip install`, `cargo add`, `go get`
   - Git write: `git push`, `git commit`, `git add`, `git merge`, `git rebase`, `git checkout -b`
   - Dangerous flags: `sed -i`, `--force`, `--hard`

   Check the blocklist against the joined string of command + all args. Use case-insensitive matching.

3. **Path containment** â€” reject if any arg contains `..` that would escape `repoPath`, or if any arg is an absolute path that does NOT start with `repoPath`. Allow absolute paths that start with `repoPath`.

4. **Timeout** â€” pass `timeoutMs` (default 30_000) to the underlying `execCommand`. Also enforce a hard max of 60_000ms regardless of what's passed.

**On rejection:** Return a string like `"(blocked: {reason})"` â€” do NOT throw. The LLM should see the rejection and try a different approach.

**On success:** Call the existing `execCommand(cmd, args, repoPath, timeoutMs)` from `../commands.ts` and return the stdout string.

**Important:** When command is `bash` or `sh`, the actual commands are in the args (typically `["-c", "actual command here"]`). The blocklist check must also inspect the `-c` argument string for dangerous patterns.

Export `createExecCommandTool` as the sole export.

**Part B: Create `packages/audit-engine/src/tool-phase-runner.ts`**

Create a `runPhaseWithTools` function:

```typescript
export async function runPhaseWithTools(
  ctx: AuditRunContext,
  phaseNumber: number,
): Promise<void>
```

Implementation:
1. Get `repoContext` string via `getRepoContext(ctx.auditId)`
2. Get `model` via `getModel(ctx, phaseNumber)`
3. Get `guideChunk` via `getGuideChunk(phaseNumber)`
4. Build prompt via `buildToolUsePhasePrompt(guideChunk, repoContext, ctx.repoPath, FINDING_FORMAT_TEMPLATE)`
5. Create the execCommand tool via `createExecCommandTool(ctx.repoPath)` â€” use default 30s timeout
6. Call `generateText` from "ai" with:
   - `model`
   - `prompt` (the built prompt)
   - `tools: { execCommand: execCommandTool }` â€” single tool
   - `output: Output.object({ schema: PhaseOutputSchema })` â€” structured output matching existing schema (PRF-13)
   - `stopWhen: stepCountIs(15)` â€” cap at 15 tool call rounds to prevent infinite loops
   - `maxOutputTokens: 16384` â€” enough for findings JSON
7. Extract the structured output from the result. In AI SDK v6 with `output`, the result has `.output` property containing the parsed object.
8. Extract token usage from `result.usage` (has `inputTokens`, `outputTokens` â€” renamed in v6 from promptTokens/completionTokens). Compute totalTokens.
9. Build `findings` array, set `phase: phaseNumber` on each finding (same as runPhaseLlm does).
10. Build `outputMd` string: `# Phase ${phaseNumber} â€” ${getPhaseName(phaseNumber)}\n\n${output.summary}\n\n## Findings\n${JSON.stringify(findings, null, 2)}`
11. Write outputMd to `${ctx.auditOutputDir}/phase-${String(phaseNumber).padStart(2, "0")}.md`
12. Call `markPhaseCompleted(ctx.auditId, phaseNumber, outputMd, findings, totalTokens)`

Import `fs` from "node:fs/promises" and `path` from "node:path" for file writing.
